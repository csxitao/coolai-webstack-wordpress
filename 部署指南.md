# CoolAI WebStack WordPress 部署指南

本文档说明如何把当前项目部署到 WordPress。当前项目已集成 JustNews 主题页面模板，并使用 Must-Use Plugin 提供 REST API 和后台管理能力。

## 一、上传文件

### 1. 后端与后台文件

上传本地目录：

```text
mu-plugins/
```

到服务器目录：

```text
/wp-content/mu-plugins/
```

最终应包含：

```text
/wp-content/mu-plugins/coolai-integration.php
/wp-content/mu-plugins/coolai-admin.php
/wp-content/mu-plugins/views/dashboard.php
/wp-content/mu-plugins/views/bookmarks.php
/wp-content/mu-plugins/views/my-bookmarks.php
/wp-content/mu-plugins/views/favicons.php
/wp-content/mu-plugins/views/logs.php
/wp-content/mu-plugins/views/settings.php
```

说明：`mu-plugins` 是 WordPress Must-Use Plugin 目录，文件放入后会自动加载，不需要在后台手动启用。

### 2. 前端页面和页面模板

上传本地目录：

```text
themes/justnews/
```

到服务器主题目录：

```text
/wp-content/themes/justnews/
```

最终应包含：

```text
/wp-content/themes/justnews/coolai-nav.html
/wp-content/themes/justnews/page-coolai.php
```

## 二、推荐更新方式

如果只是前端样式、卡片、菜单、手机端、tooltip、搜索体验等变化，通常只需要替换：

```text
/wp-content/themes/justnews/coolai-nav.html
```

如果涉及 WordPress API、同步、favicon、后台管理，则同步替换：

```text
/wp-content/mu-plugins/coolai-integration.php
/wp-content/mu-plugins/coolai-admin.php
/wp-content/mu-plugins/views/
```

如果涉及页面模板入口，则替换：

```text
/wp-content/themes/justnews/page-coolai.php
```

## 三、访问与验证

部署后建议逐项检查：

1. 访问导航页面，确认页面正常打开。
2. 切换分类，确认左侧分类和右侧卡片正常。
3. 测试本地收藏搜索和搜索引擎跳转。
4. 测试新增、编辑、删除网址。
5. 测试批量选择、批量移动、批量删除。
6. 测试文件夹/集合卡片和文件夹内卡片信息提示。
7. 测试大卡片、小卡片、暗黑模式、背景主题。
8. 测试手机端分类栏和卡片显示。
9. 登录 WordPress 后测试云同步。
10. 测试 JSON 导入/导出。

## 四、后台入口

部署后，WordPress 后台会出现 CoolAI 相关管理入口，主要用于：

- 查看仪表盘
- 管理书签数据
- 查看我的书签
- 管理 favicon 缓存
- 查看日志
- 调整设置

## 五、故障排查

| 问题 | 建议处理 |
|------|----------|
| 页面空白 | 检查 `coolai-nav.html` 是否上传到当前启用主题的 `justnews` 目录 |
| 页面模板无效 | 检查 `page-coolai.php` 是否在当前主题目录 |
| API 报错 | 检查 `mu-plugins` 文件是否完整上传 |
| 后台菜单缺失 | 检查 `coolai-admin.php` 是否在 `/wp-content/mu-plugins/` |
| 同步失败 | 确认 WordPress 登录状态、REST API 地址和 nonce |
| 图标不显示 | 清理浏览器缓存，或在后台/favicon 功能中刷新图标 |
| 导入失败 | 确认 JSON 中包含分类和网站数据，当前版本已兼容旧格式 |
| 手机端错位 | 清理缓存后重新访问，确认上传的是最新 `coolai-nav.html` |

## 六、上线前建议

- 先备份当前服务器上的 `coolai-nav.html`。
- 新版在测试路径确认无误后再替换正式文件。
- 每次上线后记录 GitHub 提交号，方便回滚。
