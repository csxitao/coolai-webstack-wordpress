# CoolAI WebStack WordPress 技术文档

## 一、整体架构

```text
前端：themes/justnews/coolai-nav.html
  - WebStack 左右布局
  - 分类、网址卡片、文件夹、搜索、批量操作
  - localStorage 状态与用户偏好
  - 调用 WordPress REST API

页面模板：themes/justnews/page-coolai.php
  - 注入 WordPress 登录态、REST API 地址、nonce 等配置
  - 加载 coolai-nav.html

后端：mu-plugins/coolai-integration.php
  - REST API
  - 用户数据同步
  - favicon 代理/缓存
  - 网页标题和图标抓取

后台：mu-plugins/coolai-admin.php + views/
  - 仪表盘
  - 书签管理
  - favicon 管理
  - 日志与设置
```

## 二、前端状态

核心状态位于 `BookmarkApp` 的 `this.state`，主要包含：

```javascript
{
  categories: {},
  websites: {},
  categoryOrder: [],
  currentCategory: 'ai',
  currentView: 'grid',
  sortMode: false,
  batchMode: false,
  lockCategories: true,
  sidebarCompact: false,
  sidebarAutoHide: false,
  localSearchQuery: '',
  showCounts: true,
  showVisitStats: true
}
```

同时使用 `visitStats` 记录访问统计，使用 icon cache 记录 favicon 加载结果。

## 三、数据存储

| 数据 | 位置 | 说明 |
|------|------|------|
| 分类 | localStorage / WordPress user_meta | 分类 ID 与名称 |
| 网站 | localStorage / WordPress user_meta | 按分类组织的网址数据 |
| 分类排序 | localStorage / WordPress user_meta | 左侧分类顺序 |
| 设置 | localStorage | 视图、显示、侧边栏、暗黑模式等 |
| 访问统计 | localStorage | 每个网站访问次数 |
| favicon | WordPress 上传目录/媒体库/缓存 | 图标代理、缓存、手动上传 |

## 四、REST API

命名空间：

```text
coolai/v1
```

主要端点：

| 方法 | 端点 | 权限 | 功能 |
|------|------|------|------|
| POST | `/wp-json/coolai/v1/sync` | 登录 | 保存同步数据 |
| GET | `/wp-json/coolai/v1/sync?key=xxx` | 登录 | 获取指定同步数据 |
| GET | `/wp-json/coolai/v1/sync/status` | 登录 | 同步状态 |
| GET | `/wp-json/coolai/v1/sync/all` | 登录 | 获取全部同步数据 |
| GET | `/wp-json/coolai/v1/favicon?domain=xxx` | 公开 | favicon 代理与缓存 |
| GET | `/wp-json/coolai/v1/fetch-meta?url=xxx` | 登录 | 获取网页标题、描述和图标 |

## 五、关键交互

### 卡片 tooltip

普通卡片和文件夹内卡片都使用 `.has-tooltip` + `data-tippy-content`。当前版本已统一提示内容为：

- 网站名
- 网址
- 描述

并处理了：

- 左右边缘不超出屏幕
- 靠近侧边栏不被遮挡
- 文件夹弹窗内 tooltip 层级和边界
- 空描述时自动兜底显示名称/网址

### 搜索

搜索包含两类：

- 本地收藏搜索：过滤自己收藏的网址。
- 搜索引擎跳转：输入关键词后跳转到当前选择的搜索引擎。

### 批量操作

批量模式支持：

- 选择多个网址
- 移动到其他分类
- 删除选中网址
- 删除整个分类
- 搜索结果内批量选择

### 侧边栏模式

桌面端支持：

- 标准模式
- 窄模式
- 自动隐藏模式
- 自动隐藏时浮动分类菜单

移动端会改为顶部横向分类栏。

### 文件夹/集合

网址可以放入文件夹。文件夹内支持：

- 打开/关闭
- 文件夹内卡片展示
- 文件夹内排序
- 移出文件夹
- 文件夹内卡片 tooltip

## 六、安全与兼容

- 前端输出使用 `escapeHtml` 和 `escapeJsString` 降低 XSS 风险。
- URL 会进行协议校验，避免危险协议。
- 后端 REST API 使用登录态、nonce 和权限校验。
- 导入功能兼容旧格式和当前格式。
- favicon 获取失败时有默认图标兜底。

## 七、文件维护原则

当前项目中只保留根目录一份文档：

```text
README.md
部署指南.md
技术文档.md
```

`coolai-webstack-preview/` 只保留代码副本，不再放重复 md 文档。

前端调整后，需要保持这两份一致：

```text
coolai-webstack-preview/themes/justnews/coolai-nav.html
themes/justnews/coolai-nav.html
```
