## 一、项目架构

### 1.1 整体设计

```
┌─────────────────────────────────────────────────────────────┐
│                    前端 (coolai-nav.html)                    │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │  UI 组件层  │  │  状态管理   │  │  数据持久化层        │  │
│  │  - 分类导航 │  │  - state    │  │  - localStorage     │  │
│  │  - 书签卡片 │  │  - 视图模式 │  │  - WordPress API    │  │
│  │  - 模态框   │  │  - 同步状态 │  │                     │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
└───────────────────────────┬─────────────────────────────────┘
                            │ REST API
┌───────────────────────────▼─────────────────────────────────┐
│             后端 (coolai-integration.php)                   │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │  API 路由   │  │  权限验证   │  │  数据存储            │  │
│  │  /sync      │  │  - 登录检查 │  │  - user_meta        │  │
│  │  /favicon   │  │  - 频率限制 │  │  - 媒体库 (图标)    │  │
│  │  /fetch-meta│  │  - Nonce    │  │                     │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 数据流

```
用户操作 → 更新 state → 保存 localStorage → 触发云同步 → REST API → WordPress数据库
                ↑                                               │
                └───────────────── 拉取同步 ←──────────────────┘
```

### 1.3 存储设计

| 存储位置 | 数据类型 | 说明 |
|---------|---------|------|
| localStorage | categories | 分类列表 `{id: name}` |
| localStorage | websites | 网站数据 `{categoryId: []}` |
| localStorage | categoryOrder | 分类排序 |
| localStorage | settings | 用户偏好设置 |
| WordPress user_meta | coolai_* | 云端同步数据 |
| WordPress 媒体库 | favicon | 网站图标永久存储 |

## 二、前端技术详解

### 2.1 状态管理

```javascript
this.state = {
    categories: {},        // 分类数据
    websites: {},          // 网站数据（按分类组织）
    categoryOrder: [],     // 分类排序
    currentCategory: 'ai', // 当前选中分类
    viewMode: 'compact',   // 视图模式
    lockCategories: true,  // 分类锁定状态
    syncStatus: 'idle',    // 同步状态
    lastSyncTime: null     // 最后同步时间

```

### 2.2 核心方法

| 方法 | 功能 | 触发时机 |
|------|------|---------|
| `init()` | 初始化应用 | 页面加载 |
| `loadData()` | 加载本地数据 | 初始化时 |
| `renderWebsites()` | 渲染网站列表 | 数据变更时 |
| `saveWebsites()` | 保存网站数据 | 增删改操作后 |
| `syncToServer()` | 同步到服务器 | 登录用户操作后 |
| `fetchWebsiteInfo()` | 获取网站元信息 | 添加网站时 |

### 2.3 事件绑定策略

```javascript
// ✅ 推荐：使用事件委托
document.addEventListener('click', (e) => {
    if (e.target.matches('.website-card')) {
        // 处理点击
    }
});

// ✅ 动态元素：创建后立即绑定
const categoryTab = document.createElement('div');
this.bindCategoryContextMenu(categoryTab);

// ❌ 避免：重复绑定导致内存泄漏
element.addEventListener('click', handler); // 多次调用会叠加
```

### 2.4 安全处理

```javascript
// HTML 转义（防止 XSS）
escapeHtml(text) {
    const div = document.createElement('div');
    div.textContent = text;
    return div.innerHTML;
}

// JavaScript 字符串转义（用于 onclick 属性）
escapeJsString(str) {
    return str.replace(/[\\'"]/g, '\\$&')
              .replace(/\n/g, '\\n')
              .replace(/\r/g, '\\r');
}

// URL 安全验证
sanitizeUrl(url) {
    // 多层解码防止绕过
    let decoded = url;
    while (decoded !== decodeURIComponent(decoded)) {
        decoded = decodeURIComponent(decoded);
    }
    // 检查危险协议
    if (/^(javascript|data|vbscript):/i.test(decoded)) {
        return '#';
    }
    return url;
}
```

## 三、后端 API 设计

### 3.1 端点列表

| 方法 | 端点 | 权限 | 功能 |

| GET | `/wp-json/coolai/v1/sync/status` | 登录 | 同步状态 |

## 四、WordPress 开发注意事项

### 4.1 Must-Use Plugin 特性

```
位置：/wp-content/mu-plugins/
特点：
- 自动加载，无需激活
- 无法在后台停用
- 比普通插件更早加载
- 不支持子目录（需要loader）
```

**适用场景**：
- 核心功能插件
- 安全相关功能
- 多站点必需功能

### 4.2 REST API 开发规范

```php
// ✅ 正确：使用 register_rest_route
register_rest_route('coolai/v1', '/sync', array(
    'methods' => 'POST',
    'callback' => array($this, 'save_data'),
    'permission_callback' => array($this, 'check_permission'),  // 必须提供
    'args' => array(
        'key' => array(
            'required' => true,
            'type' => 'string',
            'sanitize_callback' => 'sanitize_text_field'  // 输入清理
        )
    )
));

// ❌ 错误：遗漏 permission_callback
register_rest_route('coolai/v1', '/sync', array(
    'methods' => 'POST',
    'callback' => array($this, 'save_data')
    // WordPress 5.5+ 会警告
));
```

### 4.3 用户数据存储

```php
// 使用 user_meta 存储用户数据（数据隔离）
$user_id = get_current_user_id();

// 保存
update_user_meta($user_id, 'coolai_categories', $data);

// 读取
$data = get_user_meta($user_id, 'coolai_categories', true);

// 删除
delete_user_meta($user_id, 'coolai_categories');
```

**注意**：
- user_meta 值会自动序列化/反序列化
- 大数据量考虑使用自定义表
- 敏感数据需要加密存储

### 4.4 Transient API（临时数据/缓存）

```php
// 设置临时数据（带过期时间）
set_transient('coolai_rate_limit_' . $user_id, $count, 60);

// 获取
$count = get_transient('coolai_rate_limit_' . $user_id);

// 删除
delete_transient('coolai_rate_limit_' . $user_id);
```

**适用场景**：
- API 频率限制计数
- 缓存外部 API 响应
- 临时数据存储

### 4.5 媒体库文件存储

```php
// 将图片保存到媒体库
$upload_dir = wp_upload_dir();
$file_path = $upload_dir['path'] . '/' . $filename;
```

### 4.6 错误处理

### 5.2 后端优化

- **分页查询**：大数据量分页处理
- **索引优化**：自定义表添加索引

### 5.3 请求优化

```javascript
// 请求间隔控制
const REQUEST_INTERVAL = 500; // 毫秒

// 指数退避重试
async function fetchWithRetry(url, options, maxRetries = 3) {
    for (let i = 0; i < maxRetries; i++) {
        try {
            const response = await fetch(url, options);
            if (response.status === 429 || response.status === 503) {
                await sleep(1000 * Math.pow(2, i));
                continue;
            }
            return response;
        } catch (error) {
            if (i === maxRetries - 1) throw error;
            await sleep(1000 * Math.pow(2, i));
        }
    }
}
```
